Auth Middleware
src/middleware/
authenticate auth.js
Parses Authorization: Bearer <token> header. Verifies JWT with jsonwebtoken. On success, attaches req.user = { userId, username, isAdmin }. Returns 401 if missing, expired, or invalid — all cases return the same message.
ensureSelf auth.js
Ownership guard. Compares req.user.userId from JWT against req.params.userId or req.body.userId. Allows access on match, or if req.user.isAdmin === true. Returns 403 on mismatch. All protected user/transaction routes use this.
authorizeAdmin auth.js
Role guard. Checks req.user.isAdmin === true. Must run after authenticate. Returns 403 if user is not an admin. Only applied to GET /api/users.
validateBody validate.js
Generic validator factory. Checks that specified fields exist and are not null/empty in req.body. Returns 400 with a list of missing fields. Pre-built validators: validateRegister, validateLogin, validateResetPassword, validateTransaction.
Global middleware: requestLogger logs every request before routing. handleMulterError catches file upload errors (size, type). All error responses use { "success": false, "message": "..." }.
